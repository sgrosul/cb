# Первопричины ошибок и внесённые изменения:
1. Ошибка кэширования npm

Первопричина: В workflow указан cache-dependency-path: package-locks.json, но фактический файл называется package-lock.json (без 's')

Исправление: Изменён путь на package-lock.json (строка 24)

Дополнительно: Версия Node.js обновлена с 20 на 24 из-за предупреждения о депрекации

2. Отсутствие скрипта lint:ci

Первопричина: В package.json нет скрипта lint:ci, есть только lint

Исправление: Заменён npm run lint:ci на npm run lint (строка 30)

3. Неверная точка входа для smoke check

Первопричина: Сборка создаёт файл dist/index.js, а не dist/main.js

Исправление: Изменён путь на node dist/index.js (строка 36)

4. Ошибка в пути артефакта при деплое

Первопричина: В команде деплоя указан artifacts, но артефакт скачивается в папку artifact

Исправление: Изменён путь на artifact в команде деплоя (строка 54)

# Отчёт
## Что изменено:
В ходе отладки CI/CD pipeline были исправлены четыре критических ошибки: некорректный путь к файлу кэша npm (package-locks.json → package-lock.json), отсутствующий скрипт lint:ci (заменён на lint), неверная точка входа при проверке сборки (main.js → index.js) и несоответствие пути артефакта при деплое (artifacts → artifact). Также обновлена версия Node.js с 20 на 24 для соответствия актуальной среде выполнения GitHub Actions. Все изменения затронули исключительно инфраструктурную часть — файл .github/workflows/ci-cd.yml.

## Как проверено:
Локально выполнена команда npm ci && npm run verify — все проверки проходят успешно. В CI каждый шаг был последовательно исправлен и перезапущен для подтверждения корректности. Итоговый запуск workflow в PR завершился со статусом Success по всем обязательным job, артефакты успешно передаются между этапами.

## Какие ограничения остались:
Pipeline привязан к конкретной версии Node.js (24) и может потребовать обновления при её депрекации. Smoke check выполняется только для одного файла, без проверки целостности всего собранного пакета. В процессе деплоя отсутствует проверка успешности скрипта deploy.sh, что может скрыть потенциальные ошибки развёртывания. Также отсутствует тестовое окружение для проверки деплоя перед отправкой в staging.
